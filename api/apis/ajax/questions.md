# AJAX - Interview Questions

## Questions (1-10)

### 1. What is AJAX?
**Answer:** Asynchronous JavaScript and XML. Technique for updating web pages without full reload using JavaScript and XMLHttpRequest.

### 2. How does AJAX work?
**Answer:** JavaScript creates XMLHttpRequest, sends request to server, processes response, updates DOM without page reload.

### 3. What is XMLHttpRequest?
**Answer:** Browser API for making HTTP requests from JavaScript. Enables asynchronous communication with server.

### 4. What is the difference between synchronous and asynchronous AJAX?
**Answer:** Synchronous blocks until response. Asynchronous continues execution, handles response in callback.

### 5. How does Fetch API differ from XMLHttpRequest?
**Answer:** Fetch is modern, promise-based, cleaner API. XMLHttpRequest is older, callback-based, more complex.

### 6. What are the advantages of AJAX?
**Answer:** Better user experience, faster interactions, reduced server load, partial page updates, no page reload.

### 7. What are the disadvantages of AJAX?
**Answer:** Browser history issues, SEO challenges, cross-domain restrictions, JavaScript required, debugging complexity.

### 8. What is CORS? How does it relate to AJAX?
**Answer:** Cross-Origin Resource Sharing. Allows AJAX requests to different domains if server allows.

### 9. How do you handle errors in AJAX requests?
**Answer:** Check status codes, handle network errors, use try/catch with async/await, implement retry logic.

### 10. What is the difference between GET and POST in AJAX?
**Answer:** GET for retrieving data (parameters in URL). POST for submitting data (in request body). Same as regular HTTP.
