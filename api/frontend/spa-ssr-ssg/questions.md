# SPA/SSR/SSG - Interview Questions

## Questions (1-10)

### 1. What is a SPA (Single Page Application)?
**Answer:** Web app that loads single HTML page, updates content dynamically via JavaScript. No full page reloads.

### 2. What are the advantages of SPA?
**Answer:** Better UX, faster navigation, offline capability, reusable APIs, modern development experience.

### 3. What are the disadvantages of SPA?
**Answer:** Initial load time, SEO challenges, browser history complexity, JavaScript required, memory management.

### 4. What is SSR (Server-Side Rendering)?
**Answer:** Rendering HTML on server, sending complete HTML to client. Each request generates HTML on server.

### 5. What are the advantages of SSR?
**Answer:** Better SEO, faster initial load, works without JavaScript, social media sharing, better for content sites.

### 6. What is SSG (Static Site Generation)?
**Answer:** Pre-rendering pages at build time. Generate HTML files, serve as static files. No server-side rendering per request.

### 7. When would you use SPA vs SSR vs SSG?
**Answer:** SPA: interactive apps, dashboards. SSR: dynamic content, SEO important. SSG: blogs, documentation, marketing sites.

### 8. What is hydration in SSR?
**Answer:** Process of making server-rendered HTML interactive. JavaScript loads, attaches event handlers, takes over.

### 9. What is ISR (Incremental Static Regeneration)?
**Answer:** Static pages regenerated on demand or schedule. Combines SSG benefits with dynamic updates. Next.js feature.

### 10. How do you choose between SPA, SSR, and SSG?
**Answer:** Consider: interactivity needs, SEO requirements, content type, update frequency, infrastructure, user experience goals.
