# C4 Model - Interview Questions

## Questions (1-8)

### 1. What is the C4 Model?
**Answer:** Visual notation for software architecture. Four levels: Context, Container, Component, Code. Different levels of detail.

### 2. What are the four levels of C4 Model?
**Answer:** Context (system and users), Container (applications/services), Component (within container), Code (classes/modules).

### 3. What is Context diagram?
**Answer:** Highest level, shows system and external actors (users, external systems). What system does, who uses it.

### 4. What is Container diagram?
**Answer:** Applications, databases, file systems. Shows high-level technical choices, how containers communicate.

### 5. What is Component diagram?
**Answer:** Components within container. Shows internal structure, responsibilities, how components interact.

### 6. What is Code diagram?
**Answer:** Classes, interfaces, relationships. Detailed implementation level, UML-like, usually auto-generated.

### 7. Why use C4 Model?
**Answer:** Clear communication, different audiences, scalable abstraction, easy to understand, standard notation.

### 8. How do you create C4 diagrams?
**Answer:** Tools: Structurizr, PlantUML, draw.io, Miro. Start with Context, drill down as needed, keep simple.
