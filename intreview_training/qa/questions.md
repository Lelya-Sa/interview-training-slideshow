# QA (Quality Assurance) - Interview Questions

## Questions (1-10)

### 1. What is QA? What is the difference between QA and testing?
**Answer:** QA (Quality Assurance): process ensuring quality, prevention. Testing: finding defects, verification. QA is broader.

### 2. What are types of testing?
**Answer:** Unit, Integration, E2E, Functional, Performance, Security, Usability, Regression, Smoke, Load testing.

### 3. What is the difference between manual and automated testing?
**Answer:** Manual: human executes tests, flexible, time-consuming. Automated: scripts run tests, fast, repeatable, requires maintenance.

### 4. What is test-driven development (TDD)?
**Answer:** Write tests before code. Red-Green-Refactor cycle: write failing test, write code to pass, refactor.

### 5. What is the testing pyramid?
**Answer:** Many unit tests (fast, cheap), some integration tests, few E2E tests (slow, expensive). Foundation is unit tests.

### 6. What is regression testing?
**Answer:** Testing to ensure changes don't break existing functionality. Re-run previous tests, automated regression suite.

### 7. What is integration testing?
**Answer:** Testing how components work together. API testing, database integration, service integration, end-to-end flows.

### 8. What is performance testing?
**Answer:** Testing system performance under load. Load testing, stress testing, capacity testing, measure response times.

### 9. What is test coverage?
**Answer:** Percentage of code executed by tests. Line coverage, branch coverage, statement coverage. Aim for high coverage.

### 10. How do you write good tests?
**Answer:** Clear test names, one assertion per test, arrange-act-assert pattern, test edge cases, independent tests, fast tests.
