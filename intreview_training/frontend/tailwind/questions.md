# Tailwind CSS - Interview Questions

## Questions (1-10)

### 1. What is Tailwind CSS?
**Answer:** Utility-first CSS framework. Low-level utility classes, build custom designs without leaving HTML.

### 2. What is utility-first CSS?
**Answer:** Approach using small, single-purpose utility classes. Compose designs from utilities, no custom CSS needed.

### 3. What are the advantages of Tailwind?
**Answer:** Rapid development, consistent design, small bundle size (purge unused), no naming conflicts, responsive design.

### 4. What are the disadvantages of Tailwind?
**Answer:** HTML can be verbose, learning curve, requires build process, less semantic HTML, team adoption needed.

### 5. How does Tailwind's purge work?
**Answer:** Removes unused CSS in production. Scans files, keeps only classes used. Reduces bundle size significantly.

### 6. How do you customize Tailwind?
**Answer:** tailwind.config.js file. Customize colors, spacing, fonts, breakpoints, add custom utilities, plugins.

### 7. What is the difference between Tailwind and Bootstrap?
**Answer:** Tailwind: utility-first, compose designs, no components. Bootstrap: component-based, pre-built components, more opinionated.

### 8. How do you handle responsive design in Tailwind?
**Answer:** Use responsive prefixes: sm:, md:, lg:, xl:, 2xl:. Mobile-first approach, breakpoints defined in config.

### 9. What are Tailwind plugins?
**Answer:** Extend Tailwind with custom utilities, components, variants. Community plugins, or create custom ones.

### 10. When would you use Tailwind vs traditional CSS?
**Answer:** Tailwind: rapid prototyping, design systems, utility approach. Traditional: complex animations, very custom designs, team preference.
