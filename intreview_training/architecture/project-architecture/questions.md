# Project Architecture - Interview Questions

## Questions (1-10)

### 1. What is project architecture?
**Answer:** Structure and organization of codebase. How code is organized, modules, folders, components, dependencies.

### 2. What are common project structures?
**Answer:** Feature-based, layer-based, modular monolith, domain-driven, MVC-based structures.

### 3. What is feature-based structure?
**Answer:** Organize by features (user, product, order). Each feature has own folder with all related code.

### 4. What is layer-based structure?
**Answer:** Organize by layers (controllers, services, models, repositories). Separation by technical concerns.

### 5. How do you organize a Node.js project?
**Answer:** Routes, controllers, services, models, middleware, utils, config. Separate concerns, use dependency injection.

### 6. How do you organize a React project?
**Answer:** Components, pages, hooks, services, utils, constants, types. Feature folders or component-based organization.

### 7. What is modular monolith?
**Answer:** Monolith organized as modules. Each module is independent, clear boundaries, can extract to microservice later.

### 8. How do you manage dependencies in project architecture?
**Answer:** Dependency injection, interface-based design, avoid circular dependencies, clear dependency direction.

### 9. What is the importance of project structure?
**Answer:** Maintainability, scalability, team collaboration, onboarding, code organization, easier testing, clear boundaries.

### 10. How do you evolve project architecture?
**Answer:** Start simple, refactor as needed, extract modules, introduce patterns gradually, document decisions, review regularly.
