# API - Interview Questions

## Questions (1-10)

### 1. What is an API?
**Answer:** Application Programming Interface - set of rules/protocols for building software. Defines how components interact.

### 2. What are the types of APIs?
**Answer:** REST, GraphQL, SOAP, gRPC, WebSocket, RPC. Public, private, partner APIs.

### 3. What is API design?
**Answer:** Process of planning and creating APIs. Includes endpoints, data formats, authentication, versioning, documentation.

### 4. What are API best practices?
**Answer:** RESTful design, versioning, consistent naming, proper status codes, error handling, documentation, rate limiting.

### 5. What is API versioning?
**Answer:** Managing changes to API. Methods: URL (/v1), headers, query parameters. Maintain backward compatibility.

### 6. What is API documentation?
**Answer:** Documentation explaining how to use API. Includes endpoints, parameters, responses, examples, authentication.

### 7. What is API rate limiting?
**Answer:** Restricting number of requests client can make. Prevents abuse, ensures fair usage, protects backend.

### 8. What is API authentication?
**Answer:** Verifying client identity. Methods: API keys, OAuth, JWT tokens, Basic Auth. Prevents unauthorized access.

### 9. What is the difference between public and private APIs?
**Answer:** Public: accessible to anyone, documented, rate limited. Private: internal use, restricted access, higher rate limits.

### 10. What is API testing?
**Answer:** Testing API functionality, performance, security. Methods: Unit tests, integration tests, load testing, security testing.
